# Architecture Decision Records

Architecture Decision Records (ADRs) capture consequential technical choices and the reasoning that made them appropriate at the time. Add one when a decision meaningfully constrains future implementation or would otherwise be difficult to reconstruct.

Use the naming convention:

```text
0001-short-decision-title.md
```

Do not create an ADR for routine implementation details.

## Template

```markdown
# Decision title

## Status

Proposed | Accepted | Superseded | Deprecated

## Context

What problem, constraint, or trade-off requires a decision?

## Decision

What was decided and why?

## Consequences

What becomes easier, harder, required, or intentionally limited?

## Alternatives considered

What credible alternatives were evaluated, and why were they not selected?
```
